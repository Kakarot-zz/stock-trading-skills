# 妙想模拟交易 API 调试笔记

## API Key 权限模型

同一个 MX_APIKEY 有两种权限层级：

| 权限 | 可用端点 | 失败返回 |
|------|----------|----------|
| 只读（查询） | `/positions`, `/balance`, `/orders` | — |
| 交易 | `/trade`, `/cancel` | `code=114` "API密钥不存在或已失效" |

**重要**：返回 114 不一定意味着 Key 过期，也可能是 Key 只有只读权限没有交易权限。
如果读操作成功但交易失败，首先确认该 Key 是否在东方财富妙想 Skills 页面开启了交易权限。

## Trade API 字段映射

```
POST https://mkapi2.dfcfs.com/finskillshub/api/claw/mockTrading/trade
```

| 字段 | 说明 | 示例 |
|------|------|------|
| `type` | `buy` 或 `sell` | `"sell"` |
| `stockCode` | 6位股票代码 | `"002380"` |
| `quantity` | 股数（整数） | `10000` |
| `useMarketPrice` | 是否市价 | `false` |
| `price` | 价格（放大取整） | `40500` |

**price 放大规则**：
- 上交所（6/9开头）：价格 × 100（2位小数）
- 深交所（0/3开头）：价格 × 1000（3位小数）

例：卖出 002380 收盘45.00元，跌停40.50元
```
decimal_places = 3  # 深交所
price_int = int(round(40.50 * 1000))  # = 40500
```

**常见报错**：
- `code=100, message="输入价格不合理，超过限价"` — 价格超出涨停/跌停限制，或精度错误
- `code=501, message="可用余额不足"` — 账户可用资金不够
- `code=114` — Key 无交易权限（参见上方权限模型）

## 新浪实时价格 API（push2 被封时的备选）

当 `push2.eastmoney.com` 被防火墙封堵时，用新浪行情接口获取实时价格：

```python
import urllib.request

url = 'https://hq.sinajs.cn/list=sz002380,sz301591,sz000509,sz002129,sz300491,sh600726,sh603032'
req = urllib.request.Request(url, headers={'Referer': 'https://finance.sina.com.cn'})
with urllib.request.urlopen(req, timeout=8) as r:
    data = r.read().decode('gbk')
```

返回格式：`var hq_str_sz002380="科远智慧,44.660,44.670,45.000,46.610,44.060,45.000,45.020,..."`

字段顺序（以深市为例）：
```
0: 股票名称
1: 今日开盘价
2: 昨日收盘价
3: 当前价格（最新价）
4: 今日最高
5: 今日最低
6: 买一价
7: 卖一价
8: 成交量（股）
9: 成交额（元）
10: 买一量
11: 买一价
...以此类推
```

**注意**：
- 深市前缀 `sz`，沪市前缀 `sh`
- 北交所 `bj`
- 返回 GBK 编码，需 `decode('gbk')`

## mx_moni.py 脚本限制

`extract_trade_info` 函数提取价格的逻辑是取**最后一个数字**作为价格：

```python
# 最后一个数字会被当作价格
# "卖出 002380 41.00 10000股" → price=10000（错误！）
# "卖出 002380 价格 40.50 10000股" → price=40.50（正确，因为"价格"关键词）
```

正确用法：
```bash
# 标准格式
python mx_moni.py "卖出 002380 价格 41.00 10000股"
python mx_moni.py "买入 600519 1700 100"
python mx_moni.py "市价买入 000002 1000"
```
