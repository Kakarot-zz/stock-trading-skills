## 核心方法（3-tier，优先级顺序）

lhb['代码'] = lhb['代码'].astype(str).str.zfill(6)

## 上榜原因分类表

| 上榜原因关键词 | 含义 |
|--------------|------|
| 日涨幅偏离值达到7% | 单日10cm涨幅异常，需出龙虎榜 |
| 连续三个交易日内，涨幅偏离值累计达到20% | 主板连续3日异动 |
| 日涨幅达到15% | 创业板/科创板20cm涨停 |
| 连续三个交易日内收盘价格涨幅偏离值累计达到30% | 科创/创业板连续3日涨30% |
| 日换手率达到20% | 高换手率上榜 |
| 日价格振幅达到15% | 日内振幅大 |
| 北交所3日涨跌幅累计达±40% | 北交所特规 |
| 有价格涨跌幅限制的日收盘价格涨幅达到15% | 科创板涨幅15% |
| 有价格涨跌幅限制的日收盘价格涨幅偏离7% | 主板日涨幅偏离7% |
| 近期多次涨停 | 强势股反复涨停 |
| 60日新高 | 强势股突破新高 |
| 日收盘价涨幅偏离值达到7% | 主板日涨幅偏离7% |

## 常见陷阱

1. **akshare `stock_lhb_detail_em` 无席位明细** — 返回列包含上榜原因但无席位名称，席位分析需另用 MX接口
2. **函数签名** — `start_date`/`end_date`（不是 `date`），格式 `YYYYMMDD`
3. **连板股的上榜原因** — 连板股基本都有LHB记录，直接用LHB上榜原因即可
4. **北交所股票** — 上榜原因格式不同（如"北交所3日涨跌幅累计达+40%"），单独处理
5. **stock_zt_pool_strong_em 的涨停统计字段** — 是"涨停次数/成功次数"格式（如"4/4"），不是简单的连板数
6. **手动补充字典** — 板块首板关键词需根据实际板块确定，不可机械照搬历史日期的映射表
