```python
for tbl in d.get('dataTableDTOList', []):
    table = tbl.get('table', {})
    head = table.get('headName', [])
    for idx, row in table.items():
        if idx == 'headName' or not isinstance(row, list): continue
        bull_tag = row[0] if '牛散标签' in head else '-'
        amount = row[1] if len(row) > 1 else '-'
        print(f'席位={bull_tag} 金额={amount}万')
```

## 3. 预期差逆势股

**定义**：当日涨幅跑赢大盘（大盘下跌时上涨，或涨幅明显大于指数）的股票。

**⚠️ 重要限制：`stock_zt_pool_em`只返回涨停股（涨幅10%或20%），不返回涨幅3%~9.9%的强势股。**

**识别方法**：
- 对比个股涨跌幅与当日指数涨跌幅
- 涨幅>3%但非涨停，且大盘下跌>1%，视为逆势
- 重点关注：消息面无明显利好但股价强势的标的

**数据获取方式**：
1. **akshare涨停数据**中涨幅>3%的标的（已知涨停股）
2. **龙虎榜数据**中筛选涨幅排名靠前且无量化席位的标的
3. 涨停数量少（≤2只）的板块中，出现涨幅3%~9.9%的大市值股票值得关注

**实战逻辑**：大盘下跌时，逆势上涨3%~9.9%的股票比涨停股更能反映资金真实意图（涨停可能是一字板或情绪板）

## 1. 板块节点判断

### 两种运动节奏

**A. 存量博弈（涨一天跌一天）**
- 特征：板块内个股轮动，没有持续主线
- 策略：不追高，只低吸；次日冲高即走

**B. 正向运动（持续性行情）**
- 节奏：第1天起势 → 第2天全面加强 → 第3天后排分化 → 第4天局部加强
- 策略：第2天高潮后谨慎，第3天分歧后聚焦核心

### 如何判断第2天全面加强
信号1：**创业板接力切换**
- 行情连板结束后，市场短暂调整
- 创业板出现大量涨停和10cm+大涨个股
- 说明资金突然转换，普遍认可这个节奏

信号2：**弱转强数量多**
- 第1天板块内有许多烂板（尾盘勉强封住）或炸板
- 或板块有超过2个大市值个股突然放量大涨
- 以上两种情况出现→ 第2天大概率高开

**⚠️ 关键警示：第1天涨停数量就很多（≥8只）→ 第2天不要参与，板块爆发首日即高潮**

---

## 2. 逆向运动判断

### 什么是逆向运动
板块分歧日（通常在连涨2~3天后）第2天：
- 竞价大幅高开 → 但随后高开低走
- 板块内核心股走出明显的负反馈

### 逆向运动的三个阶段

**第1天（分歧日）**：
- 核心股高开低走，修复失败
- 尾盘弱势，板块整体回落

**第2天（逆向第1日）**：
- 竞价阶段要关注核心股的竞价表现
- 激进做法：**尾盘低吸核心股**
- 正常做法：**第2天做竞价量能大 + 竞价红盘的前期核心**

**第3天（逆向第2日）**：
- 如果前两天都没有像样的修复
- → 确认板块进入**退潮期**
- 后续每次修复都是逃命机会

**逆向运动持续两天无修复 → 板块退潮确定**

### 教训1：RSI低 ≠ 买入信号（最重要）

**反面案例：**
- 云南锗业 RSI=48低位 → 复盘买入 → 今日高开冲高后回落，失败
- 航天机电 RSI=29.9低位 → 复盘买入 → 今日平开低走-8.2%，止损出局

**RSI低的两种情况——截然不同的含义：**

| 场景 | RSI低的成因 | 是否可买 | 判断方法 |
|------|-----------|---------|---------|
| 上涨途中回调 | 获利盘清洗，缩量回调 | ✅ 可买 | RSI从70+快速回落至50-60，且价格回调到MA5/MA10 |
| 连续下跌趋势 | 超跌反弹（无主力控盘） | ❌ 不能买 | RSI在低位反复钝化（30~40横着走），价格不断新低 |

**通鼎互联案例（正面）：**
- 05-21跌停→RSI=34.8，但这是通信板块整体下跌中的超跌
- 05-28反包涨停→05-29继续涨+4.2%
- **关键区别**：通鼎互联有板块效应（5G/通信基建催化），有明确事件驱动；云南锗业/航天机电没有

**量化规则（必须遵守）：**
```
❌ RSI<40 + 无明确催化 + 非主线板块 = 不买
✅ RSI<40 + 有明确催化 + 板块联动 + 放量阳线 = 可买
⚠️ RSI<40 + 连板后回调（非上涨途中回调）= 观望，不买
```

### 教训2：涨停≠买入信号，炸板8次+涨停=最危险信号

**反面案例：**
- 航天机电：05-28涨停+10%但炸板8次 → 今日平开后单边下跌-8.2%
- 云南锗业：05-28涨停+10%（龙虎榜10.1亿净买）→ 今日冲高97后回落，失败

**量化规则：**
```
❌ 炸板次数≥8 + 涨停 = 封板质量极差，不买
❌ 炸板次数≥8 + 昨日涨停 = 次日开盘即走，不格局
✅ 炸板次数0~2 + 涨停 = 封板质量好，可格局
```

### 教训3：高开≠持有信号，需盯盘30分钟判断

**反面案例：**
- 云南锗业：开盘高开+5.8% → 10:00前冲高97 → 单边回落至90 → 尾盘最低89.05
- 这是典型"高开拉高出货"，不是"高开强势"

**分时判断三问法（买入后持有多看）：**

| 问题 | 强走（持有） | 弱走（卖出/止损） |
|------|------------|----------------|
| 开盘30分钟能创日内新高吗？ | ✅ 不断创新高 | ❌ 高开后无法新高 |
| 回调时能守住分时均线吗？ | ✅ 回调不破均线 | ❌ 均线压制下行 |
| 成交量是缩量还是放量？ | 缩量=锁仓，强势 | 放量=分歧，出货 |

**量化规则：**
```
高开后30分钟无法创日内新高 → 减仓50%
高开后放量下跌 + 收盘接近最低 → 次日止损
```

### 教训4：龙虎榜净买入≠次日上涨

**反面案例：**
- 云南锗业：龙虎榜净买10.1亿（国泰海通自贸区+武汉）→ 今日高开后直接回落

**核心逻辑：**
- 龙虎榜反映的是**昨日收盘前**的资金意图
- 主力可以在龙虎榜公布后**次日直接平仓**
- 净买入≠锁仓持有

**量化规则：**
```
✅ 净买入 + 涨停封死（无炸板）= 次日溢价概率高
⚠️ 净买入 + 炸板多次 = 主力可能已在对倒，次日谨慎
❌ 净买入 + 高开低走 = 主力借高开出货，不买
```

### 教训5：解套博弈区（高位被套盘密集区）= 最大压力位

**云南锗业案例：**
- 05-12涨停建仓的主力被套在100元区间
- 05-26跌到80元超跌
- 05-28反弹涨停（自救）
- 05-29冲到97→正好是05-12的解套区→立刻被砸

```
✅ 股价距离前高>10%+ 成交量萎缩 = 突破概率大，可买
⚠️ 冲高至前高附近 + 放量回落 = 主力解套出货，立刻止损
```

## 常见陷阱

1. **akshare日期格式** — 必须`'YYYYMMDD'`，不能用`'YYYY-MM-DD'`
2. **龙虎榜席位必须用MX接口** — akshare `stock_lhb_detail_em` 只有汇总无席位明细，datacenter API（RPT_BILLBOARD_DAILYDETAILSBUY）已失效（code=9501）。必须用 MX接口 `client.query('{代码} 龙虎榜')` 获取席位牛散标签来做量化过滤。
3. **龙虎榜API pageSize=300 不保证返回300条** — 实际数量由服务器决定（实测有时只返回28条），需要先收集全部 rows 再按 `SECURITY_CODE` 聚合。
4. **龙虎榜API `SECURITY_NAME_ABBR` 字段不存在** — 直接用 `SECURITY_CODE` 查名称，通过腾讯行情 API `https://qt.gtimg.cn/q=sh600330,sz000063` 批量获取。
5. **龙虎榜数据去重陷阱** — 遍历已排序 rows 后直接 set() 去重 → 同一只股票只取第一条，后面记录被跳过。正确做法：先全部收集 rows → 按 `SECURITY_CODE` 聚合求 NET → 最后排序。
6. **量化席位频率限制** — 每次席位查询间隔≥2.5秒
7. **DDX频率限制** — 每次DDX查询间隔≥3秒
8. **创业板涨停** — akshare涨停数据中筛选`代码.str.startswith('300')`
9. **板块高潮阈值** — 涨停≥5只为高潮，非≥3只
10. **逆向运动退潮** — 持续2天无修复才确认退潮，不要第一天就下结论
11. **push2.eastmoney.com被封** — `stock_sector_fund_flow_rank`、`stock_board_industry_name_em`等全部失败，改用涨停数量估算板块热度
12. **一字板=封死是误解** — 一字板可能炸板（如中京电子炸4次），需同时看炸板次数和连板数
13. **修复封板≠尾盘涨停** — 早盘炸板后尾盘重新封住按原首次封板时间分类，不算真正的尾盘涨停
14. **`stock_zt_pool_em`只含涨停股** — 不返回3%~9.9%涨幅的强势股，预期差逆势股需结合龙虎榜数据综合判断
15. **持仓数据源优先级（2026年5月实测）** — MX接口`{name} 收盘价`/`{name} 涨跌幅` > 腾讯K线前复权收盘价 > 腾讯实时行情涨跌幅字段。腾讯实时字段在收盘后严重失真（误差可达15%+），严禁用于持仓数据
16. **持仓分析中的量化席位判断逻辑** — "连板股高开要卖"的决策依据不是RSI等技术指标，而是席位属性。量化基金/量化打板/拉萨标签=无格局，次日必砸；价值游资=有格局，可持有。实证：香江控股5连板但有量化打板席位，清仓正确

17. **腾讯行情[46]字段涨停价不可直接使用** — 2026-06-02实测：利通电子603629的fields[46]=22.84，但实际股价195元，正确涨停价应为216.66×1.10=238.33。推算涨停价必须用 `昨收[4]×1.10` 公式，字段[46]只作为参考。

18. **腾讯行情成交额字段[35]单位是万元** — fields[35]格式为`价格/成交量/成交额`，其中成交额单位是万元不是元。用 `price × volume` 可以交叉验证：195.41×408981=7991万（≈8000万），不等于fields[35]拆出的816307万。成交额异常大时需用换手率交叉验证。

19. **持仓代码存疑时的处理** — 持仓成本与现价偏差>30%或代码在行情API查不到时（如利通电子原代码603812），立即标注⚠️存疑，不得用推测数据填充。正确做法：查询正确代码（603629），用腾讯行情逐字段解析核实。

20. **RSI/均线数据标注规范** — 子代理返回的RSI数据未经独立验证不得标注"已核实"，必须标注"⚠️待核实"。腾讯K线API收盘后返回空数据时，在报告开头注明RSI/均线⚠️待核实。

### ⚠️ 持仓个股走势描述的日期校验规则（2026-05-29新增，最重要）

**错误现象**：把一只股票的涨停历史（05-28涨停）错误地安在另一日期（05-29）上，导致复盘报告 factual error。

**根本原因**：`stock_zt_pool_em(date='YYYYMMDD')` 返回的是"**当天**有哪些股票涨停"，不是"某只股票在**哪天**涨停"。

**正确流程（强制步骤）**：
```
Step 1：用 stock_zt_pool_em(date='target') 确认目标日期涨停池
Step 2：检查持仓股是否在当日涨停池中
Step 3：若不在 → 当日没有涨停，用 MX日线数据描述走势
Step 4：若在 → 从涨停池取涨停数据（封板时间/炸板次数等）
```

**实证**：
- 002636在05-28涨停（+9.99%，封板11:25:27，炸板3次）
- `stock_zt_pool_em(date='20260529')` 返回49只涨停池，002636不在其中
- 05-29正确描述：收盘53.51元（+0.02%），高位平盘，**不是"涨停炸板"**

**禁用做法**：不能因为"05-28涨停"就直接写"05-29涨停炸板"，必须先查05-29涨停池确认。

### ⚠️ 持仓"是否涨停"的交叉验证规则（2026-06-02新增）

**实战教训**：06-02复盘把华塑控股(+4.67%)和通鼎互联(+8.15%)均描述为"今日涨停"——错误。两只都不在06-02涨停池67只中，实际涨幅未达涨停价。

**根本原因**：先入为主地认为"持仓涨幅大=涨停"，没有用涨停池做第一步验证。

**强制交叉验证流程**：
```
Step 1：用 stock_zt_pool_em(date='today') 拿到当日涨停池（代码集合）
Step 2：持仓股代码是否在涨停池中？
    → 在 → 涨停，取涨停池字段（封板时间/炸板次数/连板数）描述
    → 不在 → 非涨停，用 MX日线涨幅 描述走势（如"+8.15%涨幅，通信板块联动"）
Step 3：禁止跳过Step1直接凭涨跌幅判断"涨停"
```

**⚠️ 容易踩坑的场景**：
- 持仓涨幅+8%但不在涨停池 → 写"今日涨幅+8.15%，非涨停，通信板块跟风"
- 持仓昨日首板今日继续涨 → 必须先确认今日涨停池里有没有它，再判断连板数
- 龙虎榜因"日振幅15%"上榜 ≠ 涨停（如华塑控股06-02振幅15%上榜，但涨幅仅+4.67%）

### 持仓分析中的量化席位判断逻辑（2026-05-29修订）

**"连板股高开要卖"的决策依据不是RSI等技术指标，而是席位属性。**

| 席位类型 | 特征 | 操作策略 |
|---------|------|---------|
| 量化基金/量化打板 | 近三月买入>300次，标签含"量化"字样 | 无格局，次日必卖，不格局 |
| 拉萨/T王 | 席位名称含拉萨/T王/互联网营业部 | 量化激进，次日砸盘 |
| 价值游资（炒股养家/章盟主） | 无量化标签，历史买入次数合理 | 有格局，可持有等高位 |
| 技术指标高位（RSI等） | 结果指标，非操作依据 | ⚠️ RSI高≠必然下跌，板块情绪才是关键 |

**实证**：香江控股5连板，RSI持续高位，但"量化打板席位操盘"才是"高开即走清仓"的判断依据，5连板后清仓正确。单纯看RSI会得出错误结论（"RSI高所以要卖"——但如果板块情绪强，量化游资可以继续拉板）。
